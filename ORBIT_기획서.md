# Orbit — 기획서

> 버전: v0.1 | 작성일: 2026-04-21 | 상태: 기획 중

---

## 1. 제품 개요

### 제품명
**Orbit** — 모든 업무가 프로젝트를 중심으로 돈다

### 한 줄 설명
개인과 팀의 프로젝트·할일·회의록을 하나의 앱에서 관리하는 로컬 우선 업무 관리 도구

### 핵심 문제 정의

| 현재 문제 | Orbit의 해결 방향 |
|-----------|-----------------|
| 노션: 강력하지만 일상 할일 보기가 복잡함 | 심플한 UI, 클릭 2번 이내로 핵심 작업 완료 |
| 지라: 기능이 많지만 무겁고 세팅이 어려움 | 로컬 설치, 인터넷 없이도 즉시 사용 |
| 개인 업무 ↔ 회사 업무가 분리되지 않음 | 워크스페이스 단위로 완전 분리 |
| 모바일 접근이 불편함 | Phase 3에서 모바일 앱 지원 |

### 타겟 사용자

| 단계 | 대상 |
|------|------|
| Phase 1 | 개인 개발자, 프리랜서, 1인 기업 |
| Phase 2 | 소규모 팀 (2~20인), 스타트업 |
| Phase 3 | 개인 + 회사 동시 사용자 |

## 2. 핵심 개념

### 2-1. 워크스페이스 (Workspace)
- 개인 / 회사A / 회사B ... 단위로 분리
- 각 워크스페이스는 독립된 프로젝트·멤버·데이터 보유
- 전환은 사이드바 상단 드롭다운으로 1클릭

### 2-2. 프로젝트 (Project)
- 모든 업무의 최상위 단위
- 제목, 설명, 색상, 마감일, 상태(진행중/완료/보류)
- 할일·회의록·문서가 프로젝트에 소속

### 2-3. 할일 (Task)
- 기존 TodoStick의 할일 시스템 계승
- 프로젝트에 연결(선택) 또는 독립 할일
- 담당자, 우선순위, 태그 추가 (Phase 2)

### 2-4. 회의록 (Meeting Note)
- 날짜 + 제목 + 내용 (마크다운 지원)
- 프로젝트에 연결
- 회의에서 나온 액션아이템 → 할일로 즉시 변환

### 2-5. 대시보드 (Dashboard)
- 오늘 할일 요약
- 프로젝트별 진행률
- 미완료 이월 항목

## 4. 기능 명세

### Phase 1 — 로컬 단독 (현재 ~ v1.0)

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| 워크스페이스 전환 | 개인/회사 분리, 사이드바 드롭다운 | 🔴 필수 |
| 프로젝트 CRUD | 생성/편집/삭제/완료, 색상·마감일 | 🔴 필수 |
| 할일 ↔ 프로젝트 연결 | 기존 할일에 프로젝트 태그 추가 | 🔴 필수 |
| 대시보드 | 오늘 할일 + 프로젝트 진행률 요약 | 🔴 필수 |
| 회의록 작성 | 날짜·제목·내용·참석자, 프로젝트 연결 | 🟡 중요 |
| 액션아이템 → 할일 변환 | 회의록에서 할일 즉시 생성 | 🟡 중요 |
| 프로젝트 진행률 | 완료 할일 / 전체 할일 자동 계산 | 🔴 필수 |
| 스티커 프로젝트 필터 | 특정 프로젝트 할일만 스티커에 표시 | 🟢 선택 |

### Phase 2 — 팀 연동 (v2.0)

| 기능 | 설명 | 구현 방식 |
|------|------|---------|
| 팀 워크스페이스 | 여러 사람이 같은 프로젝트 공유 | 공유 JSON 파일 또는 간단한 서버 |
| 담당자 지정 | 할일에 팀원 할당 | 워크스페이스 멤버 목록 |
| 댓글 | 할일·회의록에 코멘트 | 로컬 or 서버 저장 |
| 실시간 알림 | 담당자 변경·완료 알림 | Discord Webhook 또는 자체 알림 |
| 역할 권한 | 관리자/멤버/뷰어 | 워크스페이스별 설정 |

### Phase 3 — 모바일 (v3.0)

| 기능 | 설명 |
|------|------|
| 클라우드 동기화 | 데스크탑 ↔ 모바일 실시간 동기화 |
| 모바일 앱 | React Native (iOS + Android) |
| 오프라인 지원 | 네트워크 없을 때 로컬 동작 후 재연결 시 동기화 |
| 푸시 알림 | 할일 마감, 회의 시작 알림 |

## 5. 데이터 구조

```
Workspace
├── id, name, type(personal/company), color
└── Projects[]
    ├── id, title, description, color, deadline, status
    ├── Tasks[]
    │   ├── id, title, memo, date, is_completed
    │   ├── project_id, repeat_type, repeat_days
    │   ├── priority(high/mid/low), assignee
    │   └── remind_at, color, category, order_index
    └── MeetingNotes[]
        ├── id, title, date, content (markdown)
        ├── participants[]
        └── action_items[] → Tasks로 변환 가능
```

## 6. 기술 스택

| 항목 | Phase 1 (로컬) | Phase 2 (팀) | Phase 3 (모바일) |
|------|---------------|-------------|-----------------|
| 데스크탑 | Electron + React | 동일 | 동일 |
| 데이터 저장 | JSON 파일 (현재) | 경량 서버 or 공유 파일 | Supabase |
| 모바일 | - | - | React Native |
| 팀 동기화 | - | 공유 폴더 or 미니 서버 | Supabase Realtime |
| 알림 | Discord Webhook | Discord Webhook | 푸시 알림 |
