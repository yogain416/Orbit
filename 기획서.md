# 할일 체크 앱 기획서 (TodoStick)

> 버전: v0.2 | 작성일: 2026-04-19 | 작성자: 개인 기획

---

## 1. 앱 개요

### 프로젝트 이름 (안)
**TodoStick** — 스티커처럼 붙어있는 오늘의 할 일

### 한 줄 설명
오늘 해야 할 일을 스티커 메모처럼 띄워두고, 하나씩 체크하며 완료하는 데스크탑 할일 관리 앱

### 핵심 가치
- 기존 메모앱/캘린더의 할일 확인 불편함 해소
- 당일 할 일을 항상 눈에 보이게 유지
- 월/주/일 단위로 전체적인 할일 흐름 파악

### 사용 대상
- 1단계: 개인 사용 (Windows 데스크탑)
- 2단계: 지인 배포 (설치형 앱)
- 3단계: 모바일 연동 (iOS/Android)

---

## 2. 기술 스택 선택

### 권장: Electron + React

| 항목 | 내용 |
|------|------|
| 프레임워크 | Electron (Windows 네이티브 앱) |
| UI | React + Tailwind CSS |
| 데이터 저장 | SQLite (로컬), 이후 Supabase (클라우드 동기화) |
| 모바일 연동 (이후) | React Native 또는 PWA |

### 선택 이유
- 웹 기술(React)로 개발 → 이후 모바일/웹 전환 용이
- 로컬 SQLite로 시작 → 클라우드 동기화로 확장 쉬움
- Claude Code로 개발 시 생태계 레퍼런스 풍부

## 5. 데이터 구조

### 할일 테이블 (tasks)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | 고유 식별자 |
| title | TEXT | 할일 제목 |
| memo | TEXT | 메모 (선택) |
| date | DATE | 할일 날짜 |
| is_completed | BOOLEAN | 완료 여부 |
| repeat_type | TEXT | 반복 유형 (none/daily/weekly/monthly) |
| order_index | INTEGER | 같은 날 내 순서 |
| created_at | DATETIME | 생성 시각 |
| updated_at | DATETIME | 수정 시각 |

## 6. 개발 단계 로드맵

### Phase 1: MVP — 데스크탑 핵심 기능 (예상 2~3주)

```
Week 1
  ├── Electron + React 프로젝트 세팅
  ├── SQLite 연동 및 기본 CRUD API
  └── 일별 뷰 UI 구현

Week 2
  ├── 스티커 팝업 창 구현 (항상 위에 표시)
  ├── 완료 체크 기능
  └── 할일 추가/편집/삭제 모달

Week 3
  ├── 주별 뷰 구현
  ├── 월별 뷰 구현
  └── 기본 버그 수정 및 UX 다듬기
```

### Phase 2: 완성도 향상 (예상 1~2주)

```
  ├── 반복 할일 기능

  ├── Windows 알림 연동
  └── 설치 패키지 빌드 (exe 설치파일)
```

### Phase 3: 모바일 연동 (예상 3~4주)

```
  ├── Supabase 계정 생성 및 DB 이전
  ├── 로그인 기능 (이메일/Google)
  ├── 데스크탑 앱 동기화 연동
```

## 8. UX 엣지케이스 정의

### 8-1. 반복 할일

| 상황 | 정의된 동작 |
|------|------------|
| 반복 할일 **삭제** | 3가지 옵션 다이얼로그: "이것만 / 이후 전체 / 처음부터 전체" |
| 반복 할일 **수정** | 2가지 옵션 다이얼로그: "이것만 / 이후 전체" |
| 반복 할일 **완료 체크** | 날짜별 독립 완료 상태 (오늘 완료 ≠ 내일 완료) |
| 반복 항목 **생성 타이밍** | 앱 실행 시 당일 날짜 체크 후 자동 생성 |

### 8-2. 날짜 경계

| 상황 | 정의된 동작 |
|------|------------|
| 미완료 할일이 날짜를 넘김 | 원래 날짜 유지 + 빨간색 강조 표시 |
| 자정 이후 앱 포커스 | 날짜 재확인 후 스티커 팝업 갱신 |
| 과거 날짜에 할일 추가 | 허용 (기록 목적) |
| 미래 날짜에 할일 추가 | 허용 (월별/주별 뷰 클릭으로 추가) |
| Windows 잠금 후 복귀 | 포커스 복귀 시 날짜 재확인 및 팝업 갱신 |

### 8-3. 스티커 팝업

| 상황 | 정의된 동작 |
|------|------------|
| 할일 0개 | "오늘은 할 일이 없어요!" 빈 상태 메시지 표시 |
| 할일 20개 이상 | 최대 높이 고정 + 내부 스크롤 |
| 팝업이 화면 밖으로 이동 | 트레이 아이콘 더블클릭 시 우측 하단으로 위치 리셋 |
| 멀티 모니터 환경 | 마지막 위치 저장, 모니터 분리 시 주 모니터 우측 하단으로 복귀 |
| 전체 완료 시 | "모두 완료! 🎉" 메시지 표시 후 팝업 유지 |
| 앱 재시작 시 | 이전 팝업 열림/닫힘 상태 기억하여 복원 |

### 8-4. 텍스트 표시 한계

| 상황 | 정의된 동작 |
|------|------------|
| 제목 입력 최대 길이 | 100자 제한 + 남은 글자 수 카운터 표시 |
| 일별 뷰 제목 표시 | 2줄까지 표시, 초과 시 `...` 처리 |
| 주별 뷰 제목 표시 | 8자 + `...` |
| 메모 표시 | 모달 최대 높이 고정 + 내부 스크롤 |

### 8-5. 완료 체크 / 실수 방지

| 상황 | 정의된 동작 |
|------|------------|
| 실수로 완료 체크 | 5초 토스트 메시지 + "취소" 버튼 (Undo) |
| 할일 삭제 | 확인 없이 즉시 삭제 + 5초 Undo 토스트 |
| 스티커 팝업 체크 | 메인 화면과 실시간 양방향 동기화 |

### 8-6. 뷰 전환 및 네비게이션

| 상황 | 정의된 동작 |
|------|------------|
| 주별 뷰 월 경계 | 날짜 기준 표시, 월 표기 병기 (예: 4/28~5/4) |
| 월별 뷰 오늘 날짜 | 파란 원으로 하이라이트 |
| 월별 뷰 날짜 클릭 | 해당 날짜 일별 뷰로 전환 |
| 앱 첫 실행 (데이터 없음) | "첫 할 일을 추가해보세요" 온보딩 안내 |
